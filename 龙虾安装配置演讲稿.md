# OpenClaw（龙虾）安装配置 - 大白话演讲稿

> 适合用演讲方式教别人，口语化，一步一步来。

---

## 开场白

大家好！今天我跟大家分享一个特别酷的东西——**OpenClaw**，我们叫它"龙虾"。

一句话说清楚它是什么：**它是一个跑在你自己电脑上的 AI 私人助理，你可以通过 Telegram、WhatsApp、Discord 这些聊天软件跟它对话，它帮你干活。**

不是云端的，不是别人的服务器，是跑在你自己机器上的。你的数据你做主。

---

## 第一章：为什么要搞这个？

想象一下这个场景：
- 你在 Telegram 上给"龙虾"发一条消息："帮我写个 Python 脚本"
- 它直接在你电脑上写好代码，回复你
- 你说"帮我搜一下天气"，它帮你搜
- 你发张图片给它，它能看懂图片内容
- 它甚至能帮你操作浏览器、读写文件

**重点是**：你用的是你已有的聊天软件，不用装新 App。它就像一个 24 小时在线的超级助理。

---

## 第二章：安装前你需要准备什么

### 硬件要求
- 一台电脑（Windows、Mac、Linux 都行）
- 能上网

### 软件要求
- **Node.js v22 或更高版本**（这是运行环境，必须装）
- **pnpm**（包管理工具，比 npm 快）
- **Git**（用来拉代码）

### 账号和 Key
- **阿里云 DashScope API Key**（用来调用通义千问模型）
- **Telegram Bot Token**（用来连 Telegram）

> 这些东西怎么拿到？我后面一个个教。

## 第三章：安装步骤（从零开始）

### 第一步：安装 Node.js

去 Node.js 官网下载 v22 版本，装上就行。

装完之后，打开终端验证一下：
```bash
node --version
# 应该显示 v22.x.x
```

### 第二步：安装 pnpm

```bash
npm install -g pnpm
```

### 第三步：拉代码

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

或者你已经有代码了（比如我的情况），直接进目录：
```bash
cd e:/moltbot-main/moltbot-main
```

### 第四步：安装依赖

```bash
pnpm install
```

这一步会下载所有需要的库和插件，可能要等几分钟。

### 第五步：构建项目

```bash
pnpm build
```

把 TypeScript 代码编译成可运行的 JavaScript。

### 第六步：初始化配置

```bash
pnpm openclaw onboard
```

这会启动一个交互式向导，引导你完成基本配置。

## 第四章：配置详解（最重要的部分）

配置文件是一个 JSON 文件。我来一块一块讲清楚我都配了什么。

### 4.1 AI 模型配置（大脑）

```
"龙虾用什么大脑来思考？"
```

我用的是阿里云的**通义千问**（Qwen），通过 DashScope API 调用。

配了三个模型，各有分工：

| 模型 | 定位 | 特点 |
|------|------|------|
| **qwen-max** | 主力模型 | 最聪明，能看图+文字，32K 上下文 |
| **qwen-plus** | 性价比款 | 也能看图，131K 超长上下文，便宜 |
| **qwen-turbo-latest** | 快速款 | 只处理文字，最快最便宜 |

**关键配置说明：**
```json
{
  "baseUrl": "https://dashscope.aliyuncs.com/compatible-mode/v1",
  "apiKey": "你的DashScope API Key",
  "api": "openai-completions"
}
```

- `baseUrl`：指向阿里云 DashScope 的接口地址
- `apiKey`：去阿里云 DashScope 控制台申请
- `api: "openai-completions"`：用 OpenAI 兼容模式，这样龙虾就能直接对接

**怎么拿 DashScope API Key？**
1. 去阿里云官网注册账号
2. 搜索"DashScope"或"百炼平台"
3. 开通服务，在控制台找到 API Key
4. 复制粘贴到配置文件里

### 4.2 Agent 配置（助理的行为设定）

```json
"agents": {
  "defaults": {
    "model": {
      "primary": "dashscope/qwen-max"   // 默认用最强的模型
    },
    "workspace": "E:\\moltbot-workspace", // 工作目录
    "maxConcurrent": 4,                   // 最多同时处理 4 个任务
    "sandbox": { "mode": "off" },         // 关闭沙盒（让它能直接操作你电脑）
    "subagents": { "maxConcurrent": 8 }   // 子任务最多 8 个并行
  }
}
```

**大白话翻译：**
- **主力模型**选的 qwen-max，最聪明的那个
- **工作目录**设在 `E:\moltbot-workspace`，龙虾干活的文件都放这
- **沙盒关了**，意思是它可以直接在你电脑上执行命令、读写文件（注意安全！）
- **最多同时干 4 件事**，子任务可以开到 8 个

### 4.3 记忆搜索配置（长期记忆）

```json
"memorySearch": {
  "provider": "openai",
  "model": "text-embedding-v3",
  "remote": {
    "baseUrl": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "apiKey": "你的Key"
  }
}
```

这个是让龙虾有"记忆"。它会把重要信息存起来，下次聊天还能记住。用的是 DashScope 的文本向量模型来做语义搜索。

### 4.4 Telegram 连接配置（通讯通道）

```json
"channels": {
  "telegram": {
    "enabled": true,                    // 开启 Telegram
    "botToken": "你的Bot Token",        // Telegram 机器人令牌
    "dmPolicy": "pairing",             // 私聊需要配对验证
    "allowFrom": ["*"],                // 允许所有人
    "groupPolicy": "allowlist",        // 群聊用白名单
    "streamMode": "partial",           // 流式输出（打字机效果）
    "timeoutSeconds": 120,             // 超时 2 分钟
    "retry": {                         // 重试机制
      "attempts": 5,                   // 最多重试 5 次
      "minDelayMs": 500,              // 最短等 0.5 秒
      "maxDelayMs": 30000             // 最长等 30 秒
    }
  }
}
```

**怎么拿 Telegram Bot Token？**
1. 打开 Telegram，搜索 `@BotFather`
2. 发送 `/newbot`
3. 按提示给机器人起个名字
4. BotFather 会给你一串 Token，长得像 `8391602187:AAECAS6h8Bli...`
5. 复制粘贴到配置里

**dmPolicy 为什么选 "pairing"？**
- 安全考虑！别人不能随便跟你的龙虾聊天
- 第一次要用配对码验证身份
- 验证过一次之后就不用再验证了

### 4.5 网关配置（控制中心）

```json
"gateway": {
  "port": 18789,          // 监听端口
  "mode": "local",        // 只在本机运行
  "auth": {
    "mode": "token",
    "token": "moltbot-local-token-2026"  // 访问密码
  }
}
```

网关是龙虾的"总调度室"。所有消息进来、出去都经过这里。
- 端口 `18789`，你可以通过 `http://localhost:18789` 访问控制面板
- `mode: "local"` 表示只在本机跑，外面访问不了（安全）

### 4.6 语音配置（会说话的龙虾）

```json
"tts": {
  "auto": "always",           // 每次回复都自动生成语音
  "provider": "edge",         // 用微软 Edge 的语音引擎
  "edge": {
    "voice": "zh-CN-XiaoxiaoNeural",  // 晓晓的声音（中文女声）
    "lang": "zh-CN",
    "outputFormat": "audio-24khz-48kbitrate-mono-mp3"
  }
}
```

配了之后，龙虾每次回复不光发文字，还会发一条语音消息！用的是微软 Edge 浏览器内置的 TTS 引擎，免费的，声音还挺自然。

### 4.7 浏览器配置（能上网冲浪的龙虾）

```json
"browser": {
  "enabled": true,
  "profiles": {
    "clawd": {
      "color": "#FF4500",    // 橙红色标识
      "cdpPort": 18800       // Chrome 调试端口
    }
  },
  "defaultProfile": "clawd"
}
```

开了之后，龙虾可以控制 Chrome 浏览器帮你：
- 打开网页
- 截图
- 填表单
- 抓取信息

### 4.8 工具权限配置（能干什么不能干什么）

```json
"tools": {
  "allow": ["*"],    // 允许所有工具
  "elevated": {
    "enabled": true,
    "allowFrom": { "telegram": ["*"] }  // Telegram 来的请求有高级权限
  },
  "sandbox": {
    "tools": {
      "allow": ["exec", "process", "read", "write", "edit",
                "browser", "image", "download", "speech",
                "tts", "stt", "glob", "search", "memory", "web"]
    }
  }
}
```

这里列出了龙虾能用的所有"工具"：
- `exec` / `process`：执行命令
- `read` / `write` / `edit`：读写编辑文件
- `browser`：操作浏览器
- `tts` / `stt`：文字转语音 / 语音转文字
- `search` / `web`：搜索和上网
- `memory`：记忆存取

### 4.9 增强配置里的系统提示词

在 enhanced-config.json 里还加了一段系统提示词，告诉龙虾它是谁、能干什么：

```
你是一个智能助手。你会主动帮助用户完成任务，而不是只回答问题。

你的能力包括：
1. 文件操作：可以读取、编辑、创建文件
2. 代码分析：可以分析代码结构、找出问题
3. 执行命令：可以在用户的电脑上执行命令
4. 网络搜索：可以搜索网络获取信息
5. 浏览器控制：可以自动化浏览器操作
6. 记忆功能：会记住重要信息供以后参考
```

## 第五章：启动龙虾

配置都搞好之后，启动很简单：

```bash
# 开发模式（改代码会自动重启）
pnpm gateway:watch

# 或者正式启动
pnpm openclaw gateway --port 18789
```

启动成功后你会看到日志输出，然后：
1. 打开 Telegram
2. 找到你创建的那个机器人
3. 发一条消息试试
4. 第一次会让你输入配对码（在终端日志里能看到）
5. 配对成功后就可以正常聊天了！

## 第六章：我搞了几个配置版本

在摸索过程中，我搞了三个版本的配置文件：

| 文件名 | 说明 |
|--------|------|
| **stable-config.json** | 稳定版，用了 Google Gemini 2.0 Flash 模型（免费），双模型配置 |
| **new-config.json** | 精简版，只用 DashScope/Qwen，干净利落 |
| **enhanced-config.json** | 增强版，加了系统提示词、技能配置、更多工具权限 |

**演进路线：**
```
stable（试水）→ new（精简）→ enhanced（全功能）
```

稳定版我试了 Gemini，后来觉得千问更适合中文场景，就切到了 DashScope。增强版在精简版基础上加了系统提示词和技能列表。

## 第七章：整体架构一句话总结

```
你的手机 Telegram
      ↓ 发消息
Telegram 服务器
      ↓ 推送
你电脑上的龙虾 Gateway（18789 端口）
      ↓ 调用
DashScope API（通义千问大模型）
      ↓ 返回结果
龙虾处理 → 回复到 Telegram
      ↓ 你看到回复
```

就是这么简单：**消息进来 → AI 处理 → 消息出去**。

## 第八章：常见问题 & 踩过的坑

### Q: Telegram 连不上怎么办？
A: 检查网络环境，可能需要代理。配置里有 `network.autoSelectFamily: true` 就是为了解决 IPv4/IPv6 的问题。

### Q: 模型回复太慢？
A: 换 qwen-turbo-latest，最快最便宜。或者把 maxConcurrent 调小。

### Q: 沙盒关了安全吗？
A: 如果只有你自己用，问题不大。但如果要给别人用，建议打开沙盒 `"mode": "docker"`。

### Q: 为什么配了重试机制？
A: 网络不稳定的时候，消息可能发送失败。配了 5 次重试、最长等 30 秒，基本能覆盖大部分情况。

### Q: 能不能换别的模型？
A: 当然！OpenAI GPT、Anthropic Claude、Google Gemini 都支持，改配置文件就行。

## 结尾

总结一下，装一个龙虾你需要做这几件事：

1. **装环境**：Node.js + pnpm
2. **拉代码**：git clone
3. **装依赖**：pnpm install && pnpm build
4. **搞 Key**：去阿里云拿 DashScope API Key
5. **搞 Bot**：去 Telegram 找 BotFather 创建机器人
6. **写配置**：把 Key 和 Token 填进 JSON 配置文件
7. **启动**：pnpm gateway:watch
8. **配对**：在 Telegram 里完成配对验证
9. **开聊**！

就这么多，没什么神秘的。核心就是一个 Node.js 服务 + 一个 AI 模型 + 一个聊天通道，三个东西连起来就完事了。
