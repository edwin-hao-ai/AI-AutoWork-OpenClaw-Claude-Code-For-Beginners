## 坑 #1：AI 记不住你的规矩⏱ 约 5 分钟

> [PPT参考] 第11张幻灯片：对话框截图模拟——第一次说了语言要求，第二次打开新对话又要重复

【讲师提示】这个坑几乎100%的人踩过。讲完停顿，说"有过这个经历的，聊天框扣个'踩'"，看到反应多再继续，加强代入感。

**场景还原：**

你今天打开 ChatGPT，说："以后回复我用中文，语气正式，按结论-原因-建议这个格式来。"

AI 说："好的，我记住了。"

然后你们聊了一个小时，很顺利。

第二天，你打开 AI——新对话，新开始。

你问一个问题，AI 给你回了英文，格式乱了，语气也变了。

**你得重新说一遍那段规矩。**

下周，同样的事再发生一次。

**这个坑的本质是什么？**

Chat 模式的 AI 没有"持久记忆"。每次打开新对话，它是一张白纸，上次的规矩全忘了。

你每次用 AI，都要把规矩重新喂一遍——这本质上是在把你的时间浪费在"重复培训 AI"上。

---

**解法：AGENTS.md = AI 员工手册**

> [PPT参考] 第12张幻灯片：一份文档图片 + "写一次，永久生效"

你有没有给新员工发过入职手册？

手册里写着：公司规矩、工作流程、汇报格式——新员工看完，照做，不用你每天叮嘱。

AGENTS.md 就是你给龙虾写的"入职手册"。

你把规矩写进这个文件，龙虾每次启动，自动读取这份手册，永远按你的规矩来。

**职场真实案例：**

李姐是一家公司的销售助理，每天要让 AI 整理客户跟进记录。

她在 AGENTS.md 里写了这几条：

```
1. 所有回复用中文
2. 整理客户信息时，必须按这个格式：
   - 客户名称
   - 核心需求（一句话）
   - 优先级（高/中/低）
   - 下次跟进时间
3. 语气正式，不用emoji
```

从此之后，不管她什么时候用，AI 自动按这个格式来，一次都没有跑偏。

她说："感觉 AI 真的变成我的助理了，不是随便找来聊天的网友了。"

## 坑 #2：AI 只会"说"，不会"做"⏱ 约 5 分钟

> [PPT参考] 第13张幻灯片：左边 AI 在说话，右边人在手动操作，中间画个等号

**场景还原：**

你让 AI 帮你整理一批 Word 文件，AI 说：

"好的，你可以这样做：先打开文件夹，选中所有 Word 文件，然后一个一个打开，复制第三段的内容，粘贴到一个新的汇总文档里，注意文件名要对应……"

你心想：谢了，我自己来就行，要你说什么？

这就是 Chat 模式最大的问题——**AI 是个只会出主意的顾问，不是会动手干活的工人。**

所有"说"的背后，实际操作还是你。

---

**解法：Claude Code = AI 程序员，帮你把"说"变成"做"**

> [PPT参考] 第14张幻灯片：需求输入框 → 代码生成 → 自动运行 → 结果输出，四步流程图

Claude Code 不只是"告诉你怎么做"，它自己写代码、自己运行、给你结果。

**同样的场景，用 Claude Code：**

你说："帮我把桌面上所有 Word 文件，把每个文件里第三段的内容提取出来，汇总到一个新的 Word 文档，每段前面加上原文件名。"

Claude Code 收到指令，自动：

1. 扫描桌面的所有 Word 文件
2. 打开每个文件，提取第三段
3. 生成新文档，填入内容和文件名
4. 保存到桌面，命名"汇总结果.docx"

全程你做了什么？说了一句话，等了两分钟，拿了结果。

不需要你懂任何代码，不需要你手动操作任何软件。

## 坑 #3：你和同事的 AI，说的是不同语言⏱ 约 4 分钟

> [PPT参考] 第15张幻灯片：三个人各自和 AI 聊天，输出格式各不相同，老板看得一头雾水

**场景还原：**

你们团队五个人都在用 AI 写周报。

小王让 AI 用分点列举，小李让 AI 写段落，你让 AI 按重要程度排序，小张没给任何规矩……

老板拿到五份周报，格式五花八门，信息密度不一样，挑重点都不知道从哪看起。

结果老板说："以后统一格式，按我发的模板来。"

然后你们每个人都要手动提醒自己的 AI，再次设置规矩——坑 #1 又回来了。

---

**解法：龙虾 + GitHub = 团队 AI 说同一种语言，文件实时同步**

两步解决这个问题：

**第一步：让龙虾帮你装好 git，把 AGENTS.md 推到 GitHub**

你告诉龙虾："帮我把团队的 AGENTS.md 上传到 GitHub，以后每次我修改规矩，自动同步上去。"

龙虾全部帮你搞定，不用你懂任何 git 命令。

**第二步：团队所有人的龙虾连接同一个 GitHub 仓库**

团队成员的龙虾都读同一份 AGENTS.md，所有人的 AI 按同一套规矩工作。

老板说"周报加一项'本周风险'字段"——在 GitHub 上改一行，团队所有人的龙虾下次启动自动拉取最新版本，全部生效。不用每个人单独去改。

而且不只是规矩——**工作成果也能实时同步**：你用龙虾整理好的报表，推到 GitHub，同事的龙虾自动拉取，不用微信发来发去，永远不会出现"你有最新版吗"这种问题。

**类比：**

> GitHub 是团队的共享工作台，龙虾是每个人手里操作这个工作台的机器人——任何人更新了内容，所有人的机器人都知道，立刻同步。

## 坑 #4（补充）：每次用 AI，都要重新解释背景⏱ 约 4 分钟

> [PPT参考] 第16张幻灯片：重复输入相同背景信息的对话截图

【讲师提示】这个坑是补充坑，来自真实用户反馈，讲完之后直接连解法，不用停顿太久。

**场景还原：**

你在做一个新媒体账号，每次让 AI 帮你写内容，都要先解释：

"我的账号叫XX，面向25-35岁女性，主要讲职场成长，语气亲切但有分量，不用网络流行语……"

每次都要说，每次说完才能进入正题。

有时候忘了说，AI 写出来的东西完全不对味，又要重来。

---

**解法：项目专属记忆 = 背景说一次，AI 永远记得**

在 AGENTS.md 里写上这个账号的所有背景信息，龙虾每次启动自动读取，你直接说"写一篇关于职场边界感的文章"，AI 已经知道所有背景，直接开工。
