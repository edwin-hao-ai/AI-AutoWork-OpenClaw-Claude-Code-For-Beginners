## 避坑心法 1：把随意的嘱咐，变成工程化的文档规矩  约 8 分钟

**场景还原：**
以前我们在对话框里发几百字的提示词，口头吩咐一长串要求：写文章要幽默点，加Emoji啊，别忘了分段。换个话题全没了。

**怎样合理和 AI 说话，精准操控它？**
答案是：**用 Markdown (MD) 文档来创建提示词**。

这就是我们要写的 AGENTS.md（AI 员工手册）。我们在项目根目录建一行小字文档，把你公司的业务术语、格式要求、禁忌事项清清楚楚写进去。
只要这个文档在，龙虾每次启动都会自动读取。结合我们刚才说的 Git 保险柜工作流，它的好处是：彻底避免重复问问题、重复提要求，你的规矩不仅永远保存，还能在全团队互通！

## 避坑心法 2：先 Plan 后 Auto（谋定而后动，拒绝盲盒）  约 12 分钟

> [PPT参考] 第7张幻灯片：左侧一上来就瞎干 vs 右侧先画图纸再动工，先Plan后Auto

**大家经常犯的错：**
一上来就甩个复杂任务：帮我把这些网页数据爬下来做成PPT。然后看着屏幕一阵乱跑，跑了十分钟，结果发现格式全错。不仅时间没了，还浪费了大量大模型的 Token**这就叫开盲盒。**

**核心效能心法：**
在进行复杂工作时，**强行要求 AI 先给出一个 Plan（执行计划大纲）**。

1. **先 Plan（写计划）**：你告诉龙虾：请先给出你要怎么做这件事的计划。它会列出步骤 1, 2, 3。
2. **审查与纠偏**：在这个阶段，你能一眼看穿它的逻辑。如果它理解劈叉了，在这个阶段只需说一句话纠正，成本极低。这就是纠错在行动前。
3. **后 Auto（自动执行）**：计划敲定后，再说好，开始干吧（开局 Auto）。
   
**为什么要这么做？为了节约 Token（省钱）与效率飞跃！**
让系统拿着确认好的施工图跑，就不需要一边干、一边报错、一边撤销回溯。不走冤枉路，自然速度极快又省钱。比零碎的马上开工效率要好上十倍。

**【真实世界员工工作案例】新媒体运营小王的“早会汇报”**
假设你是老板，今天把新入职的实习生小王叫进办公室，交代任务：“帮我收集今天新能源汽车行业的最新竞品情况，整理成一份Word给我，中午前要。”
> - **错误示范（一上来就瞎干 / 直接Auto开盲盒）**：小王不说二话，回去打开百度狂搜，复制了几万字的新闻扔进Word，中午交给你。结果你一看，全是没用的水军软文，你想要的蔚来、小鹏、理想的具体降价数据一个都没有。小王只能哭丧着脸重做，白干一上午（在AI世界里，这就是浪费了大量的Token和时间）。
> - **正确示范（先Plan后Auto，高效职场模式）**：小王听完任务，没有急着回座位，而是拿出会议本（产生 **Plan 计划**）：“老板，我跟您确认下接下来的具体步骤：第一步，我去乘联会官网下载上个月的新能源销量数据；第二步，去那三家头部车企官网摘抄最新的降价政策新闻；第三步，把这三家的数据做成一个简单的对比表格，并在最后写200字的总结建议。您看这个方向对吗？”
老板一看（**进入审查与纠偏纠错环节**）：“方向对，但加上比亚迪的数据，其他没问题。”
小王：“收到，我这就去办！”（**开始后Auto自动执行工作**），中午准时交出完美报告。

所以，当你使用龙虾（OpenClaw）和 Claude Code 时，**你就是那个发任务的老板，AI就是你的实习生小王**。
永远记得：花10秒钟让你的超级AI实习生先把工作步骤（Plan）列出来给你看，你点完头，再让它滚去干活（Auto）！这将为你节省无数的时间！
