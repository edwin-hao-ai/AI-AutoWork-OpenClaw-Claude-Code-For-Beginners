## 模块 1：怎样像用网盘一样用好 Git（保险柜）？ 约 10 分钟

> [PPT参考] 第9张幻灯片：Git 降维打击图解：本地保险柜 + 云端共享

别害怕命令行，教大家极简工作流（其实 VS Code 编辑器里点点鼠标就能做）：

**第一步：本地存档。**你想好了一个做表规则，写进了 AGENTS.md。这时候点击初始化仓库，写上一句新建了做表规矩，点击提交 (Commit)。这就等于把文件存进了本地无敌保险柜。明天你不小心删了它，一秒钟就能退回去。

**第二步：云端发给全公司。**只需点击同步 (Push)，这些规矩文件就上了 GitHub 云端。（具体 Github 注册流程会后看我提供的图文手册，像注册微信一样简单）。其他同事一点拉取 (Pull)，整个公司的 AI 标准瞬间拉齐。

## 模块 2：简单工作流 vs 复杂工作流（Skill）的合理调度 约 15 分钟

我们要根据任务难度，**合理调度**我们的大脑和双手。

**第一类：简单工作流（单兵作战）**
比如需求：帮我把桌面 0820 文件夹里所有带数字前缀的图片更名为日期格式。
这种就是单点、一次性的操作，不用定复杂的长期规矩。直接在对话框发布指令，由 Claude Code 写个几行小脚本，龙虾自动执行一下，一分钟搞定。用完即毁。

**第二类：复杂工作流与 Skill（技能）调用**
比如需求：每天定时监控某个飞书群里的消息，只要产品经理发了长段反馈，就提取出核心痛点，生成表格数据并提醒我。
对于这种需求，如果纯靠大模型现场写代码解决，就会很不稳定。这里就要引入**系统内的 Skill（技能）概念**。

你可以把 Skill 理解为已经固化好的武器配件（比如飞书消息读取 Skill、飞书发消息 Skill）。
AI 通过读取文档，能知道自己系统里配了哪些 Skill。然后自动拆解任务：第一步调飞书 Skill 监控；第二步调提取分析；第三步调发送 Skill 汇报。
**这就是我们在开头说的，系统能自动调起合适的插件来做工程组合流，稳定且强悍。**
