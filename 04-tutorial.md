## 模块 1：小白用 GitHub 的 3 步⏱ 约 6 分钟

> [PPT参考] 第19张幻灯片：GitHub 首页截图，标注"注册在这里"

【讲师提示】GitHub 注册这步看似简单，但有些观众会卡在"邮箱验证"或者"全英文界面"，提前说一下"不用担心，界面全英文，但操作和注册微信一样"。

---

**第1步：注册 GitHub 账号（类比：注册微信）**

打开浏览器，输入网址：**github.com**

你会看到一个全英文的页面。不要慌，只看一个地方——右上角有一个"Sign up"（注册）按钮。

点进去，按顺序填：
1. 填你的邮箱地址（用你常用的，后面要验证）
2. 设一个密码（字母+数字，8位以上）
3. 给自己起一个用户名（比如你的名字拼音，全小写，不能有空格）
4. 点"Continue"（继续）

然后它会发一封邮件到你的邮箱，有一个验证码——复制过来填进去，就注册好了。

**整个过程和注册一个新 App 没有任何区别。**

---

**第2步：建"文件柜"（新建仓库，类比：新建文件夹）**

登录进去之后，点右上角你的头像，然后点"Your repositories"（你的仓库）。

你会看到一个绿色按钮"New"——点它。

弹出来的页面，只需要填一件事：**Repository name（仓库名字）**。

起一个你能认出来的名字，比如：
- `my-ai-assistant`（我的AI助手）
- `work-ai-rules`（工作AI规矩）
- 你自己取一个就行，不影响使用

其他所有选项，**一律不动，全部保持默认**。

拉到最下面，点绿色的"Create repository"按钮。

**恭喜，你的云端文件柜建好了。**

---

**第3步：上传你的规矩文件（类比：往微信文件传输助手发文件）**

进入刚才建好的仓库页面，你会看到一段英文提示，不用管它。

找到并点击"uploading an existing file"（上传文件）这几个字的链接。

进入上传页面，把你的 AGENTS.md 文件（我们后面会写好）**拖进去**，就和往微信传文件一样。

拖进去之后，页面最下面有个绿色按钮"Commit changes"——点它。

文件上传完成。

---

**额外：怎么邀请同事共享（类比：微信共享文件夹）**

进入你的仓库，点顶部的"Settings"（设置），左边菜单找"Collaborators"（协作者），点"Add people"，输入同事的 GitHub 用户名，邀请发出去。

同事接受邀请后，他们也能看到和使用你的规矩文件了。

## 模块 2：写你的第一份 AGENTS.md（龙虾员工手册）⏱ 约 6 分钟

> [PPT参考] 第20张幻灯片：一份 AGENTS.md 示例，左侧说明、右侧内容

【讲师提示】给观众发一份模板，让他们现场填，不要只讲不做。重点强调"大白话写就行，不需要任何格式"。顺便说一句：Claude Code 用的是 CLAUDE.md，龙虾用的是 AGENTS.md，两个文件名不一样，今天我们主要讲 AGENTS.md。

---

**核心逻辑：**

AGENTS.md 本质上就是一封写给龙虾的"上岗须知"。

你在里面写：
- 你是谁（让 AI 了解你的工作背景）
- AI 该怎么回复你（语言、格式、语气）
- 这个项目的规矩（特定任务怎么做）

**大白话写就行，不需要任何特殊格式，不需要任何编程知识。**

---

**万能填空模板（现在可以直接用）：**

打开一个文本编辑工具（Windows 的记事本、Mac 的"文本编辑"都行），新建文件，把下面的内容复制进去，改成你自己的信息：

```markdown
# 我是谁
我叫[你的名字]，在[你的公司]负责[你的工作]。

# 你是我的AI助手，请记住以下规矩

## 语言和格式
- 所有回复必须用中文
- 回复结构：先说结论，再说原因，最后给建议
- 语气：[正式/轻松/亲切，选一个]

## 这个项目是做什么的
[用一两句话描述你用这个工具干什么，比如"整理销售数据"或者"写自媒体文案"]

## 特别规矩
- [你的第一条特殊要求]
- [你的第二条特殊要求]
- [如果没有，删掉这几行就行]

## 输出格式要求
[如果你的任务需要固定格式，写在这里。比如：
每次整理客户信息，必须按这个格式：
- 客户名称：
- 核心需求：
- 跟进状态：
- 下次联系时间：]
```

改完之后，保存文件，文件名就叫 `AGENTS.md`（大写，注意后缀是 .md，不是 .txt）。

> **小提示：** 龙虾还支持两个配套文件——`SOUL.md`（定义 AI 的性格风格，比如"亲切、简洁、不废话"）和 `USER.md`（记录你的个人偏好）。今天先写 AGENTS.md 就够用了，SOUL.md 和 USER.md 以后有兴趣再探索。

---

**AGENTS.md 进阶配置：让 AI 更精准干活**

> [PPT参考] 第20.2张幻灯片：AGENTS.md 分层结构图

【讲师提示】进阶部分快速带过，不要花超过2分钟。告诉观众"这是以后想深挖再看的内容，今天先把基础版写好"。

除了基础模板，你还可以在 AGENTS.md 里加这些内容，让 AI 更精准：

**1. 禁止事项（AI 绝对不能干什么）**

```markdown
## 禁止事项
- 不要删除任何文件，除非我明确说"删除"
- 不要自动发送任何消息或邮件，操作前必须先给我看草稿
- 不要修改原始数据文件，只能新建结果文件
```

**2. 任务完成后的汇报格式**

```markdown
## 完成任务后，必须告诉我
- 做了什么（一句话总结）
- 结果保存在哪里（完整路径）
- 有没有遇到问题（有就说清楚，没有就说"一切正常"）
```

**3. 遇到不确定的情况怎么办**

```markdown
## 遇到这些情况，暂停并问我
- 找不到指定文件
- 有多个文件可能匹配，不确定选哪个
- 操作可能影响到我没有提到的其他文件
```

这三条加进去之后，AI 的行为会更"可控"——特别是涉及到文件修改、发送消息这类不可逆操作时，它会先问你再动手。

---

**三个职场填写示例：**

**示例一：销售助理**
```markdown
# 我是谁
我叫小红，在XX科技公司做销售助理，主要整理客户信息和写跟进记录。

# 规矩
- 用中文回复
- 整理客户信息必须按这个格式：客户名称/需求点/优先级(高中低)/下次跟进时间
- 不要用专业术语，客户看得懂就行
```

**示例二：自媒体运营**
```markdown
# 我是谁
我运营一个小红书账号，叫"XX的职场笔记"，面向25-35岁职场女性，
内容方向是职场成长和效率方法。

# 规矩
- 语气亲切、有温度，像朋友说话
- 每篇文章结尾必须有一个互动问题
- 标题控制在20字以内，要有数字或者具体场景
- 不用网络流行语和梗（观众偏成熟）
```

**示例三：行政管理**
```markdown
# 我是谁
我是公司行政，负责整理会议记录和发周报。

# 规矩
- 整理会议记录时，必须提取：决策事项/责任人/截止时间/下次跟进节点
- 周报格式：本周完成事项/下周计划/需要协调的资源
- 语气正式，发给老板的内容加一句"如有问题请随时告知"
```

## 模块 3：Claude Code 安装和首次使用⏱ 约 6 分钟

> [PPT参考] 第21张幻灯片：Claude Code 安装界面截图（Mac 版 + Windows 版）

【讲师提示】这步是技术门槛最高的地方，但实际上比装微信难不了多少。如果有人卡住，一定要让助手去手把手帮，不要让整场等。

---

**第1步：安装 Node.js（Claude Code 的基础环境）**

这一步听起来复杂，实际上就是"先装一个基础软件"，类比为：

> 就像你在手机上装某个 App 之前，系统提示"需要先安装XX运行环境"，你点安装就行。

打开浏览器，搜索"Node.js 官网"，或者直接输入 **nodejs.org**。

点击页面上标着"LTS"的绿色下载按钮（LTS = 稳定版，适合普通人用）。

下载完成后，双击安装包，全程点"下一步"，不用改任何设置，最后点"完成"。

安装好了。

---

**第2步：安装 Claude Code**

打开你电脑的"终端"（不要怕这个词，它就是一个可以输入文字命令的黑窗口）：

- **Mac 用户**：按 `Command + 空格`，输入"终端"，回车
- **Windows 用户**：按 `Win + R`，输入 `cmd`，回车

黑窗口出来了，不要慌，就当它是一个特殊的输入框。

在里面输入这一行（直接复制粘贴）：

```
npm install -g @anthropic-ai/claude-code
```

按回车，等它跑完（可能需要1-2分钟，会显示一堆文字，正常的）。

跑完之后，输入：

```
claude
```

按回车，如果看到欢迎信息，安装成功。

---

**第3步：连接你的 Claude 账号**

第一次启动时，Claude Code 会要求你登录 Claude 账号（就是你平时用 Claude 聊天的那个账号）。

按照屏幕提示：
1. 它会给你一个链接，复制到浏览器打开
2. 用你的 Claude 账号登录
3. 批准授权，回到终端

授权完成，Claude Code 就和你的账号绑定好了。

---

**第4步：把你的 CLAUDE.md 放到工作文件夹**

在你的电脑上建一个专门的文件夹，比如桌面上新建一个叫"AI工作助手"的文件夹。

把你刚才写好的 CLAUDE.md 放进这个文件夹里。

然后在终端里，进入这个文件夹（Mac：`cd ~/Desktop/AI工作助手`，Windows：`cd C:\Users\你的用户名\Desktop\AI工作助手`）。

再输入 `claude`，回车启动。

**Claude Code 会自动读取同一个文件夹里的 CLAUDE.md，从此按你的规矩来。**

## 模块 4：龙虾（OpenClaw）安装和首次使用⏱ 约 7 分钟

> [PPT参考] 第22张幻灯片：OpenClaw 界面截图 + 连接 Claude AI 示意图

【讲师提示】这步讲完观众会有"原来这么简单"的感觉。重点强调：龙虾通过你的手机/微信接收指令，不需要开电脑坐在那等，真正做到"躺着干活"。

---

**第1步：下载并安装 OpenClaw（龙虾）**

打开浏览器，搜索"OpenClaw AI"，找到官方网站（链接在 GitHub 仓库里）。

下载对应你系统（Mac 或 Windows）的安装包，双击安装，全程点"允许"和"下一步"。

安装完成后，龙虾会作为一个后台服务运行在你电脑上——你不用一直盯着它，它会在后台等你的指令。

---

**第2步：填入 Claude API Key（连接 AI 大脑）**

API Key 是什么？理解为"让龙虾调用 Claude AI 智能的通行证"。

去哪里拿？打开 **console.anthropic.com**，登录你的 Claude 账号，在"API Keys"页面点"创建"，复制那串字符。

打开龙虾设置，找到"AI 模型配置"，选 Claude，把 API Key 粘贴进去，点保存。

龙虾现在有了大脑。

---

**第3步：把 AGENTS.md 告诉龙虾**

把你刚才写好的 AGENTS.md 文件拖到龙虾指定的配置文件夹里（龙虾第一次启动会告诉你这个文件夹在哪）。

龙虾重启后，会自动读取你写的规矩，从此按你的员工手册来。

---

**第4步：连接你的消息平台（真正"躺着干活"的关键）**

龙虾最酷的功能：**它通过微信、企微、Telegram 等消息平台接收你的指令**——你不用开电脑，手机上发一条消息，龙虾就开始干活。

在龙虾设置里，找到"消息平台连接"，选择你用的平台（微信、企微或其他），按指引完成绑定。

绑定完成后，你用手机向指定联系人或群组发消息，龙虾就会收到并执行。

---

**第5步：测试第一个任务**

用手机微信，给龙虾绑定的联系人发一条消息：

```
帮我把桌面上的所有截图文件，移动到桌面的"截图备份"文件夹里，如果没有这个文件夹就先创建。
```

发送，等十几秒——然后看你的电脑桌面。"截图备份"文件夹出现了，截图全都移进去了。

**你用手机发了一条消息，电脑自动干完了活。全程你没有碰电脑。**

## 模块 4.5：龙虾当你的 IT 部门——3 个让人"哇"的案例⏱ 约 5 分钟

> [PPT参考] 第22.5张幻灯片：三个案例标题并排，"找软件 / 找不到就自己写 / 装好 git 自动同步"

【讲师提示】这三个案例是全场最容易引发"哇"声的地方，讲完每个案例停顿一下，等观众在聊天框反应。不需要现场演示，用描述就够了——观众能脑补画面。

---

**案例一：你再也不用自己找软件了**

以前你想装一个新软件，你要做什么？

打开百度，搜"好用的Markdown阅读器"，翻十几篇测评，找到官网，找到下载按钮，下载，安装，配置语言……半小时就没了。

现在你对龙虾说：

```
我需要一个能在电脑上好好预览 Markdown 文件的工具，
帮我找一个评价好的，下载安装好，打开给我看。
```

龙虾搜索对比，推荐 Obsidian（或 Typora），自动下载安装，打开，告诉你怎么用。

**你做了什么？说了一句话。**

---

**案例二：找不到合适的软件？龙虾自己写一个**

你对龙虾说：

```
我需要一个工具，每天早上 8 点自动把桌面上的截图，
按拍摄日期整理到不同文件夹里，比如"2026-03"这样的格式。
```

龙虾先去搜了一圈——没有完全符合你需求的现成工具。

然后它自己写了一小段代码，安装所需的运行环境，设置了定时任务，测试运行，告诉你："已设置完成，明天早上8点开始自动执行。"

**这等于龙虾给你配了一个专属的小工具，功能刚好符合你的需求，连定时器都设好了。**

你什么都没干。

---

**案例三：让龙虾装好 git，工作从此自动同步**

你对龙虾说：

```
帮我在电脑上安装 git，
把我的"客户资料"文件夹初始化成 git 仓库，
连接到我的 GitHub 账号，
以后每次我说"同步一下"，就自动提交并推送。
```

龙虾：下载 git → 安装 → 配置账号 → 初始化仓库 → 连接 GitHub → 测试推送。

从此，你每次处理完客户资料，对龙虾说"同步一下"——文件自动推送到 GitHub，

同事不用等你发微信，直接从 GitHub 拉取最新版本。

**团队协作的文件，第一次不用微信传来传去了。**

## 模块 5：现场串联 Demo——老板要报表，8 分钟搞定⏱ 约 7 分钟

> [PPT参考] 第23张幻灯片：完整流程图，5个步骤从左到右

【讲师提示】这是全场高潮，要提前测试好！Demo 如果出错不要慌，边查边解释，现实情况就是这样，观众接受度很高。提前准备好3-5个示例 Excel 文件。

---

**完整场景：**

老板今天早上发消息："把这个月5个业务组的销售数据汇总一下，算出每组环比增长，下班前发我。"

**以前的你需要：**

打开5个文件，复制数据，粘贴，算增长率，检查，整理格式……至少1.5小时。

**现在的你这样做：**

---

**第1步：确认龙虾在线（30秒）**

看一下系统托盘（Mac 右上角 / Windows 右下角），确认龙虾图标在运行，显示绿色"在线"状态。

---

**第2步：用大白话说需求（1分钟）**

在输入框里打：

```
我桌面上的"销售数据"文件夹里有5个Excel文件，分别是1组到5组的本月和上月销售数据。
帮我：
1. 读取每个文件里的销售总额
2. 计算每组的环比增长率（本月比上月增长了多少百分比）
3. 生成一个新的汇总表，第一列是组名，第二列是本月销售额，第三列是上月销售额，第四列是环比增长率
4. 把新表保存到桌面，命名为"本月销售汇总.xlsx"
```

点发送。

---

**第3步：等 AI 自动完成（5分钟）**

龙虾右侧的操作记录开始滚动——

```
正在打开文件夹...
读取 1组销售.xlsx...
读取 2组销售.xlsx...
...
计算环比增长率...
生成汇总表...
保存到桌面...
完成。
```

你去倒杯水，回来的时候，桌面上已经有一个"本月销售汇总.xlsx"了。

---

**第4步：检查结果（1分钟）**

打开汇总表，核对一眼数字，确认没问题。

发给老板。

---

**你做了什么？**

一共：说了一段话 + 等了5分钟 + 核查了1分钟。

没有打开任何 Excel，没有复制粘贴任何数字，没有手动算任何增长率。

**这就是龙虾 + AGENTS.md 的完整威力。**
