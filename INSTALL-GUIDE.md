# Claude Code 一键安装与配置指南

本文档提供了专门为 Claude Code（或其他具备自主操作能力的 AI 智能体）设计的“一键提示词”。你可以直接复制以下提示词框内的内容发送给 AI，让它自动为你完成所有相关环境的安装与配置。

## 1. 一键安装 OpenClaw 提示词

用于向 AI 下达自动下载并配置 OpenClaw 服务的指令。

> **提示词：**
> “请帮我一键安装和配置 OpenClaw 环境。OpenClaw 的参考链接是：https://github.com/OpenClaw/OpenClaw（请结合此链接或你已有的知识）。请你在终端中自动克隆代码、安装所需的依赖包，并进行必要的初始化配置。完成后，请帮我在本地启动服务，并告诉我如何验证它是否运行正常。”

## 2. 一键安装 Claude Code 提示词

在你已经打开了 VSCode，并且具备 Node.js/npm 环境的情况下使用。

> **提示词：**
> “我现在处于 VSCode 环境中，请在当前的集成终端里，帮我一键全局安装 Claude Code 命令行工具（请执行 `npm install -g @anthropic-ai/claude-code` 或最新推荐命令）。安装完成后，请告诉我如何通过 `claude auth` 等命令完成身份验证，并验证 CLI 工具是否已经可用。”

## 3. 一键匹配本地 Git 提示词及使用说明

用于让 AI 检查你的环境并帮你配好 Git 的基础设定，同时提供使用教学。

> **提示词：**
> “请检查我当前的本地 Git 状态是否正常。如果尚未配置，请帮我设置好全局的 Git 用户名和邮箱（你可以询问我想用什么信息，或者帮我生成一份示例配置命令让我修改）。配置完成后，请向我演示在这个工作区中，如何完整地执行一次：查看状态（status）、暂存（add）、提交（commit）和推送（push）操作。”

### 📌 Git 核心使用方法简要说明
如果你想手动操作，以下是日常高频 Git 命令：
- **查看文件状态**：`git status` （显示哪些文件被修改了）
- **暂存修改内容**：`git add .` （把当前目录下的所有修改放进暂存区准备提交）
- **记录提交信息**：`git commit -m "简短描述你做了什么修改"` （正式生成一条历史记录）
- **推送到云端库**：`git push` （将本地的改动同步到 GitHub/GitLab 等远程服务器）

## 4. 遇到插件或不明问题的解决指南

如果你在安装途中遇到网络中断、插件冲突、红字报错等任何状况，不要慌张，用这条提示词让 AI 帮你排雷：

> **排障提示词：**
> “我在执行【这里替换为你在做的事，比如：通过 npm 安装 Claude Code】时遇到了问题。以下是终端输出的完整报错日志：
> 【请在这里粘贴所有的报错信息】
> 请帮我分析出错的根本原因。你可以直接在终端里运行修复命令或帮我安装缺失的依赖，如果需要我手动操作（如关闭代理、重启软件等），请分步骤用最简单的语言告诉我怎么做。”

### 💡 智能体协同建议
- **充分放权**：当你在 VSCode 中使用具有终端执行权限的 AI 时，直接告诉它“**你有权限直接运行命令修复，请试着自己解决**”，这能省去大量确认步骤（已在当前项目配置为主旨）。
- **不要只发截图**：AI 对纯文本的错误日志解析最准，务必完整复制 Terminal （终端）里的文本。
